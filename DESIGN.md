# WADANG 디자인 원칙

## 제품과 사용자

WADANG은 운영자가 기간과 정원을 정해 마당을 열고, Dojang 인증 지갑이
지갑당 한 번 참여하도록 기록하는 GIWA 애플리케이션입니다.

- 운영자는 현재 캠페인의 상태, 참여 수, 종료 시각과 공유 링크를 빠르게 판단합니다.
- 참여자는 지갑·네트워크·Dojang 인증·참여 이력을 확인하고 다음 행동을 결정합니다.
- 개발자는 `isEligible`을 이용해 참여 결과와 현재 인증을 외부 접근 조건으로 사용합니다.

## 시각 언어

전통 문양을 복제하지 않고 기와의 겹침, 처마의 곡선과 와당의 원형 인장을
현대적인 화면 구조로 바꿉니다.

- 한지·백자색 바탕, 먹색, 석간주 적색, 청자·쑥색
- 자체 제작한 원형 와당 심벌
- 정보 구조를 나누는 얇은 먹색 선
- 제목의 명조 계열과 인터페이스의 고딕 계열 대비
- 인증 성공 시 짧은 인장 움직임과 `prefers-reduced-motion` 지원

색, 서체, 선과 상태 표현은 공유하지만 모든 페이지에 같은 히어로·카드·여백
구조를 반복하지 않습니다.

## 페이지 역할

| 화면 | 역할 | 밀도 | 먼저 보여줄 정보 |
| --- | --- | --- | --- |
| 홈·GASOK·덱·팀 | 제품과 제출 증거 소개 | 낮음 | 가치 제안, 작동 흐름, 검증 링크 |
| `/manage` | 반복 운영 | 높음 | 연결 지갑, 캠페인 상태, 참여 수, 종료 시각, 공유 |
| `/open` | 캠페인 생성 | 중간 | 입력값, 네트워크·지갑 상태, 불변 조건, 실행 버튼 |
| `/madang/[id]` | 참여와 상태 확인 | 중간 | 캠페인 상태, 남은 정원, 종료 시각, 현재 참여 단계 |
| `/docs` | 기술 참조 | 중간~높음 | 목차, API, 불변조건, 코드, 보안 경계, 검증 증거 |

## 문장과 용어

- 화면 제목은 현재 객체나 작업을 말합니다: `내 마당`, `마당 만들기`, `도장 확인`.
- 본문은 제목을 반복하지 않고 상태, 수치, 제약 또는 다음 행동을 보충합니다.
- `마당 열기`, `입장하기`, `마당 닫기`를 제품 동작명으로 사용합니다.
- 영문은 컨트랙트 함수, 네트워크, 주소처럼 실제 식별이 필요한 곳에만 사용합니다.
- 구현 한계와 비범위는 기술문서에 두고, 당장 판단을 바꾸지 않는 방어 문구는 작업 화면에서 제거합니다.

## 상태와 상호작용

- 지갑 미연결, 네트워크 불일치, 인증 확인 중, 미인증, 참여 가능, 참여 완료를 구분합니다.
- 시작 전, 종료, 정원 마감, 운영자 취소와 RPC 오류에 서로 다른 복구 문구를 제공합니다.
- 오류는 원인과 다음 행동을 함께 표시합니다.
- 긴 한글, 주소와 트랜잭션 해시는 컨테이너 안에서 줄바꿈합니다.
- 모바일에서도 주요 상태와 기본 행동이 설명문보다 먼저 보이고 터치 영역이 44px 이상이어야 합니다.

## 사용하지 않는 기본값

- 모든 화면을 초대형 제목과 오른쪽 설명의 동일한 split hero로 만들지 않습니다.
- 정보가 없는 영문 대문자 레이블을 장식처럼 반복하지 않습니다.
- 제품이 지원하지 않는 편집·삭제·가상 통계와 참여 기록을 화면 완성용으로 만들지 않습니다.
